<task type="auto">
  <name>Task 2: Fix coin transfer and resource handling</name>
  <files>contracts/sources/minesweeper_bet.move</files>
  <action>
    Fix the coin::transfer_from usage to work with OneChain's CoinStore:

    1. Import the correct modules:
       ```move
       use std::coin;
       use 0x2::oct;
       ```

    2. For escrow, use a simpler pattern - store payments in the GameRoom:
       ```move
       public struct GameRoom has key, store {
           ...
           player_bets: table::Table<address, u64>,
           ...
       }
       ```

    3. Instead of transfer_from (which requires CoinStore setup), use deposit pattern:
       - Player calls contract with payment as transaction argument
       - Contract deposits payment into game escrow
       - Contract holds payment until distribution

    4. Update distribute_prizes to use coin::transfer (not transfer_from):
       ```move
       coin::transfer<OCT>(game_escrow, winner_address, amount);
       ```

    5. Add timestamp using std::timestamp::now() instead of hardcoded 0

    The key insight: transfer_from requires the user to have a CoinStore resource
    at their address. If they don't, the call fails. Using transaction arguments
    bypasses this requirement.
  </action>
  <verify>

  </verify>
  <done>
    - coin::transfer_from replaced with safer pattern
    - Player bets stored in GameRoom table
